# Neural Organization v2.0 — Architecture Overview

> AI時代の企業アーキテクチャ：組織の全活動をAIが担うための設計図

---

## 設計思想

Neural Organization は、人間の**神経系**をメタファーとした企業アーキテクチャである。

従来の企業組織が「人間が実行し、ツールが補助する」モデルであったのに対し、
Neural Organization は「**AIが実行し、人間が監督する**」モデルへの転換を実現する。

### なぜ神経系メタファーか

| 性質 | 神経系 | Neural Organization |
|------|-------|-------------------|
| **並行処理** | 脳は同時に複数の処理を行う | 複数のAgentクラスタが並行動作 |
| **階層性** | 反射（脊髄）→ 本能（辺縁系）→ 思考（皮質） | 自律処理 → 定型処理 → 戦略判断 |
| **フィードバック** | 感覚→行動→結果→学習の循環 | Input→Execution→Evaluation→Learning |
| **記憶の多様性** | 短期・長期・手続き・エピソード | Working・Semantic・Procedural・Episodic |
| **恒常性** | 自律神経による体温・血圧の維持 | Autonomic Layerによるシステム安定性 |
| **可塑性** | 経験による神経回路の再編成 | 学習によるAgent構成・戦略の最適化 |

## 層間のデータフロー

### 上行フロー（Sensory → Cortex）

```
Sensory Layer → データ収集・正規化
    ↓
Memory Layer → 関連記憶の付加・コンテキスト化
    ↓
Cortex Layer → 状況分析・戦略立案・指示発行
```

### 下行フロー（Cortex → Output）

```
Cortex Layer → 戦略指示の発行
    ↓
Motor Layer → タスクの分解・実行・成果物生成
    ↓
Limbic Layer → 品質評価・コンプライアンスチェック
    ↓
Output Layer → 承認済み成果物のデプロイ
    ↓
Interface Layer → 外部世界への配信・反応受信
```

### フィードバックフロー

```
Interface Layer → 外部反応の受信
    ↓
Sensory Layer → 新たなデータとして入力
    ↓
Memory Layer → 結果を記憶（エピソード記憶として保存）
    ↓
Limbic Layer → 成果評価（KPI変化の分析）
    ↓
Cortex Layer → 戦略修正（必要に応じて）
```

## v1.0からの主要変更点

| 側面 | v1.0 | v2.0 |
|------|------|------|
| **層数** | 5層 | 7層 + Layer 0 |
| **処理モデル** | 直列パイプライン | 双方向フィードバック付き階層ネットワーク |
| **Agent数** | 2種類（Strategy + Execution） | 30+ Agent Types in 6 Tiers |
| **記憶** | 3種類（Vector + Context + KPI） | 5種類（Semantic + Episodic + Working + Procedural + Evaluative） |
| **通信** | 未定義 | MCP + A2A + Event Bus |
| **ガバナンス** | AI Critic + HITL | リスクティア別4段階 |
| **学習** | 受動的（FB保存のみ） | 3階層能動的学習 |
| **障害耐性** | 未考慮 | 冗長化 + フォールバック + 回路遮断器 |
| **外部通信** | 出力のみ | 双方向Interface Layer |
| **インフラ** | 未考慮 | Autonomic Layer |

## 設計原則

### 1. 記憶は組織資産（Memory is Organizational Capital）
組織の全知識・経験・判断基準は構造化された記憶として永続化される。
人員の入れ替わりがあっても、組織の知能は失われない。

### 2. 階層的自律（Hierarchical Autonomy）
低リスクな活動は完全自律。リスクが上がるにつれて人間の関与が段階的に増加。
「何でも人間に聞く」でも「何でもAIに任せる」でもない、**リスク比例型**の自律。

### 3. 並行分散処理（Parallel Distributed Processing）
部門別Agentクラスタは独立して並行動作する。
全てが直列で処理されるボトルネックは存在しない。

### 4. 継続的自己改善（Continuous Self-Improvement）
3階層の学習メカニズムにより、組織は日々進化する。
業務レベル → 戦略レベル → メタレベル の3段階で学習。

### 5. 観測可能性（Observability）
全てのAgent活動、意思決定、通信は記録され、追跡可能。
「なぜその判断をしたか」が常に説明可能。

### 6. 優雅な縮退（Graceful Degradation）
部分障害時でも事業は継続する。
重要度の低い機能から段階的に縮退し、コア機能を保護する。
