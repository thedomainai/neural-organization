# Neural Organization — コンセプト

> **WHY**: Neural Organization はなぜ存在するのか。何を目指すのか。

## ビジョン

Neural Organization は、AGI 時代の企業アーキテクチャである。人間が**目的と創造性**を提供し、知能が無限にスケールする組織の形を定義する。

従来の企業組織が「人間が実行し、ツールが補助する」モデルであったのに対し、Neural Organization は「**AIが知能を担い、人間が意志を注入する**」モデルへの転換を実現する。

## 核心的問い

> 人間とAIが共存する組織において、人間はどのような役割を果たすべきか？

**答え**: 人間は指示者ではなく**統治者**である。人間は「何をするか」を指示するのではなく、「なぜ存在するか」「どこへ向かうか」「何を大切にするか」という**意志**を注入する。

## 設計原理

| 原理 | 内容 |
|---|---|
| Intent over Instruction | 人間は指示ではなく意志を伝える |
| Governance, not Control | 人間は制御ではなく統治する |
| Ambient Presence | システムは環境として常に存在する |
| Radical Transparency | あらゆる推論・判断の根拠が確認可能 |
| Agency Preservation | 人間の主体性を保全する |
| Mutual Evolution | 人間とシステムが相互に高め合う |

## 人間の参与形態

| 参与形態 | 役割 |
|---|---|
| Governor（統治者） | 組織の存在理由を定義する。Purpose と Values の源泉 |
| Sensemaker（意味付与者） | データが捉えられない質的現実をシステムに翻訳する |
| Creator（創造者） | まだ存在しないものを構想する |
| Connector（接続者） | 人間同士の信頼・共感・関係性を構築する |
| Custodian（守護者） | システムの行動が価値観・規範と整合しているかを監視する |

## アーキテクチャ概要

Neural Organization は **5つのレイヤー**と**4つの横断的要素**で構成される。詳細は [framework.md](framework.md) を参照。

## v1.0 からの進化

v1.0 は「入力→記憶→処理→評価→出力」の直列パイプラインであった。v2.0 では以下が進化した:

- **双方向フィードバック**: Reflection（内省）が全層を更新する
- **横断的要素**: Purpose・Governance・Memory・Orchestration が全層に作用する
- **創発的推論**: ワークフローに従うのではなく、状態と目的から最適な行動を都度導出する
- **信頼の蓄積**: Trust Score に基づく動的な自律範囲の拡大

### v1.0 のデータフロー例（参考）

「KPI（WebサイトのPV）が下がった」というシナリオ:

1. **入力:** Analyticsツールから「PV前月比90%」というデータが入る
2. **戦略:** Strategy Agentが記憶を参照。「前回はSNS拡散が効いた」と思い出し、施策を指示
3. **実行:** Execution Agentが投稿文を作成
4. **評価1 (AI):** AI Criticがチェック → 自動修正
5. **評価2 (人):** 人間にSlackで通知。フィードバックを受ける
6. **学習:** 記憶層に「この人間は砕けた表現を好む」と追記
7. **出力:** 修正版がXに投稿される

この直列フローは v2.0 では 5 層 + 横断的要素による双方向アーキテクチャに進化している。
