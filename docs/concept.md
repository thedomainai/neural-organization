# Neural Organization — コンセプト

> **WHY**: Neural Organization はなぜ存在するのか。何を目指すのか。

## ビジョン

Neural Organization は、AGI 時代の企業アーキテクチャである。人間が**目的と創造性**を提供し、知能が無限にスケールする組織の形を定義する。

従来の企業組織が「人間が実行し、ツールが補助する」モデルであったのに対し、Neural Organization は「**AIが知能を担い、人間が意志を注入する**」モデルへの転換を実現する。

## 核心的問い

> 人間とAIが共存する組織において、人間はどのような役割を果たすべきか？

**答え**: 人間は指示者ではなく**統治者**である。人間は「何をするか」を指示するのではなく、「なぜ存在するか」「どこへ向かうか」「何を大切にするか」という**意志**を注入する。

## 設計原理

| 原理 | 内容 |
|---|---|
| Intent over Instruction | 人間は指示ではなく意志を伝える |
| Governance, not Control | 人間は制御ではなく統治する |
| Ambient Presence | システムは環境として常に存在する |
| Radical Transparency | あらゆる推論・判断の根拠が確認可能 |
| Agency Preservation | 人間の主体性を保全する |
| Mutual Evolution | 人間とシステムが相互に高め合う |

## 人間の参与形態

| 参与形態 | 役割 |
|---|---|
| Governor（統治者） | 組織の存在理由を定義する。Purpose と Values の源泉 |
| Sensemaker（意味付与者） | データが捉えられない質的現実をシステムに翻訳する |
| Creator（創造者） | まだ存在しないものを構想する |
| Connector（接続者） | 人間同士の信頼・共感・関係性を構築する |
| Custodian（守護者） | システムの行動が価値観・規範と整合しているかを監視する |

## アーキテクチャ概要

Neural Organization は **5つのレイヤー**と**4つの横断的要素**で構成される。詳細は [framework.md](framework.md) を参照。

## Day 1 — Neumann 導入初日に何が起きるか

抽象的なアーキテクチャだけでは「で、実際どうなるの？」がわからない。ここでは Neural Organization の実装プロダクトである Neumann を導入した初日に、組織で何が起きるかを記述する。

### 朝: 接続（Layer 0 Perception の起動）

CEO が Neumann の初期設定を開始する。最初のステップは Purpose の注入 — 「この組織はなぜ存在するのか」「今期どこへ向かうのか」「何を重視し、何を許容しないか」を言語化する。完璧である必要はない。Neumann は Trust Score 0.00 からスタートし、全ての判断に人間の承認を要求する。信頼はこれから築く。

続いて、日常的に使っている 3-4 のシステム（Slack、Google Workspace、CRM、GitHub 等）と接続する。Neumann は接続された瞬間から過去 30 日間のデータを取得し始める。メール、チャット、案件データ、タスク — これらは Layer 0 を通じてシグナルに変換される。

### 昼: 世界モデルの構築（Layer 1 Understanding の起動）

接続から数時間で、Neumann は Layer 1 で世界モデルを構築し始める。顧客、案件、プロジェクト、人物 — 組織内に散在していたエンティティが関係グラフとして可視化される。

この時点で CEO が最初の気づきを得る。「この顧客にはこの担当者しか接点がない」「この案件は 14 日間動きがない」——人間が感覚的に知っていたことが、構造化された形で目の前に現れる。まだ AI は何も提案しない。ただ「見えている」だけである。

### 夕方: 最初の提案（Layer 2 Reasoning → Layer 3 Execution）

世界モデルが一定の密度に達すると、Neumann は最初の提案を Slack に送る。「案件 X は 14 日間進捗がありません。フォローアップを提案します」——定型的推論（Routine Reasoning）に基づく、控えめで保守的な提案。

CEO は提案を承認するか、修正するか、却下する。**この反応そのものが最も価値のあるデータ**になる。承認すれば Neumann の Trust Score が微量上がる。修正すれば「この組織では、この状況ではこう判断する」というパターンが Layer 4 Reflection に記録される。却下すれば「ここは踏み込んではいけない領域」という境界が学習される。

### Day 1 の終わり

Day 1 の終わりに、組織が手にしているもの:

- **可視化**: バラバラだった情報が統合された世界モデル
- **最初の対話**: Neumann との最初の「提案→反応」サイクルの完了
- **学習の開始**: 人間の反応から組織固有の価値観を学ぶプロセスの起動

手にしていないもの:

- **自律性**: Neumann はまだ何も自分で判断しない。Trust Score は 0.00 付近
- **深い理解**: 世界モデルは粗い。因果モデルもまだない
- **パターン**: 組織固有の判断パターンは 1 つも蓄積されていない

Day 1 は「覚醒」の始まりである。Phase 1a（接続と信頼構築）の最初の 1 日にすぎないが、ここからシステムと組織が相互に学び合う旅が始まる。3 ヶ月後には世界モデルが成熟し、6 ヶ月後にはパターン学習が回り、12 ヶ月後には領域横断的な自律判断が可能になる — すべてはこの Day 1 の「最初の接続」と「最初の反応」から始まる。

## v1.0 からの進化

v1.0 は「入力→記憶→処理→評価→出力」の直列パイプラインであった。v2.0 では以下が進化した:

- **双方向フィードバック**: Reflection（内省）が全層を更新する
- **横断的要素**: Purpose・Governance・Memory・Orchestration が全層に作用する
- **創発的推論**: ワークフローに従うのではなく、状態と目的から最適な行動を都度導出する
- **信頼の蓄積**: Trust Score に基づく動的な自律範囲の拡大

### v1.0 のデータフロー例（参考）

「KPI（WebサイトのPV）が下がった」というシナリオ:

1. **入力:** Analyticsツールから「PV前月比90%」というデータが入る
2. **戦略:** Strategy Agentが記憶を参照。「前回はSNS拡散が効いた」と思い出し、施策を指示
3. **実行:** Execution Agentが投稿文を作成
4. **評価1 (AI):** AI Criticがチェック → 自動修正
5. **評価2 (人):** 人間にSlackで通知。フィードバックを受ける
6. **学習:** 記憶層に「この人間は砕けた表現を好む」と追記
7. **出力:** 修正版がXに投稿される

この直列フローは v2.0 では 5 層 + 横断的要素による双方向アーキテクチャに進化している。
