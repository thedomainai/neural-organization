## アーキテクチャの全体像

Neural Organization のアーキテクチャは、現実世界の情報がアウトプットに変わるまでの流れを **5つのレイヤー（L0〜L4）** で表現します。さらに、すべてのレイヤーを横断して作用する **4つの横断的要素** が、パイプライン全体に一貫性と方向性を与えます。

この「階層的なパイプライン」という構造は、私たちが恣意的に選んだものではありません。

1991年、神経科学者 Felleman & Van Essen は、霊長類の大脳皮質における視覚情報処理を網羅的にマッピングし、**分散階層処理**（Distributed Hierarchical Processing）の構造を実証しました。情報は網膜から始まり、低次の特徴検出から高次の意味理解へと、階層的に変換されていく。そして高次層からのフィードバックが低次層の処理を修正する。この「順方向の変換」と「逆方向のフィードバック」の組み合わせは、脳の情報処理の基本原理です。

軍事戦略の世界でも、同じ構造が独立に発見されています。Boyd（1986）の**OODAループ**（Observe-Orient-Decide-Act）は、観察 → 状況判断 → 意思決定 → 行動の4段階で戦場の意思決定を定式化しました。さらに、経営学者 Stafford Beer（1972）の**Viable System Model**（VSM）は、組織が生存能力を維持するために必要な5つのサブシステムを定義しています。

認知科学の分野では、Laird, Lebiere & Rosenbloom（2017）が、3つの独立した認知アーキテクチャ — ACT-R、Soar、Sigma — が40年の研究を経て共通の基本構造に収斂したことを報告しています。彼らはこれを**心の標準モデル**（Standard Model of the Mind）と名づけました。知覚モジュール、中央処理、運動出力という階層的な情報変換と、それを支える記憶システムという構造です。

Neural Organization の5層パイプラインは、これらの知見を組織設計に翻訳したものです。ただし、5層という区分そのものは設計判断であり、科学的必然ではありません。OODAは4段階、VSMは5つのサブシステムですが機能分化に基づく並行構造であり、厳密に一対一対応するわけではありません。重要なのは層の数ではなく、「階層的な情報変換」と「フィードバックによる自己修正」という構造原理が、分野を超えて繰り返し再発見されてきたという事実です。

```
 Purpose ──────────────────────────────────── Governance
 (WHY: 意志の注入)     │                      (WHO: 統治)
                       │                        │
                       ▼                        │
┌──────────────────────────────────────────────────────────┐
│  L0: Perception（知覚）                                    │
│  現実世界 → シグナル                                       │
├──────────────────────────────────────────────────────────┤
│  L1: Understanding（理解）                                 │
│  シグナル → 世界モデル                                      │
├──────────────────────────────────────────────────────────┤
│  L2: Reasoning（推論）       ◄── Purpose + Gov Gate       │
│  世界モデル × 目的 → 行動計画                                │
├──────────────────────────────────────────────────────────┤
│  L3: Execution（実行）       ◄── Governance Gate          │
│  行動計画 → アウトプット                                    │
├──────────────────────────────────────────────────────────┤
│  L4: Reflection（内省）                                    │
│  アウトプット → 学習 → 全層更新                              │
└──────────────────────────────────────────────────────────┘
       ▲                                  ▲
  Memory（記憶）                   Orchestration（統率）
  WHAT: 全層の永続状態               HOW: 層間フロー制御
```

ここからは、各レイヤーの責務と横断的要素の役割を順に見ていきます。

## 5つのレイヤー

### L0: Perception（知覚） — 現実世界 → シグナル

Perception は、組織の「感覚器官」にあたるレイヤーです。Slack や Salesforce、GitHub、財務ツール、監視システムなど、多種多様なデータソースに接続し、流入する情報からノイズを除去し、意味のあるシグナルを検出します。

ここで重要なのは、「知覚」が単なるデータ収集ではないということです。認知科学者 Edwin Hutchins（1995）は**分散認知**（Distributed Cognition）の理論において、認知は個人の頭の中に閉じたプロセスではなく、環境や道具に分散するものであることを示しました。航空機のコックピットを事例として、計器パネル、チェックリスト、乗員間のコミュニケーションが一体となって「認知システム」を構成していることを実証したのです。

組織もまた同様です。Slack のチャンネル、CRM のレコード、GitHub の PR、Datadog のアラート。これらは単なる「情報源」ではなく、組織の認知を構成する器官です。L0 が設計するのは、これらの器官を統合的に配線し、組織としての知覚能力を実現する仕組みです。

- **能動的接続**: どのデータソースに接続すべきかをシステム自身が判断します
- **シグナル検出**: ノイズから変化・異常・トレンドを抽出します
- **注意配分**: すべての情報を均等に扱うのではなく、最も重要なシグナルに注意を集中させます

### L1: Understanding（理解） — シグナル → 世界モデル

Understanding は、検出されたシグナルを統合して「生きた世界モデル」を構築・維持するレイヤーです。データベースに情報を格納するのとは本質的に異なり、顧客・商談・プロダクト・従業員といったエンティティの状態を、因果関係を含むグラフ構造として組み上げます。

このレイヤーの設計思想は、哲学者 Andy Clark & David Chalmers（1998）が提唱した**拡張認知**（Extended Mind）の仮説と共鳴します。彼らは、認知プロセスが脳の内部に閉じている必要はなく、外部の道具や記録が適切な条件を満たせば認知の一部を構成しうると論じました。組織の世界モデルは、まさにこの「拡張された認知」の実装です。個々の従業員の頭の中にある断片的な理解を、構造化された共有モデルとして外在化する。

さらに、神経科学者 Karl Friston（2010）の**自由エネルギー原理**（Free Energy Principle）は、脳の本質的な機能を「予測モデルの構築と予測誤差の最小化」として統一的に説明しました。脳は常に世界の生成モデルを維持し、入力される感覚データと予測の差分（サプライズ）を最小化するように自己を更新し続ける。L1 が構築する世界モデルも同じ原理に基づいています。現実からのシグナルが世界モデルの予測と乖離するとき、それは単なるデータの更新ではなく、「世界についての理解そのもの」の更新を駆動します。

- **エンティティモデル**: 顧客、商談、プロダクトなどの実体とその関係性を構造化します
- **因果モデル**: 「Xすれば Y が起こる、なぜなら Z だから」という因果構造を構築します
- **予測モデル**: 現在の状態から将来を確率的に予測します。予測誤差そのものが、モデル更新のトリガーになります

### L2: Reasoning（推論） — 世界モデル × 目的 → 行動計画

Reasoning は、世界モデル（L1）と人間が注入した目的（Purpose）を突き合わせ、何をすべきかを導出するレイヤーです。あらかじめ定義されたワークフローに従うのではなく、状態と目的から最適な行動を都度導出する「創発的推論」が核心です。

ノーベル経済学賞受賞者 Daniel Kahneman の**二重過程理論**（Dual Process Theory）は、人間の思考を2つのシステムに分類しました。**System 1** は高速で自動的、直感的な処理。**System 2** は低速で意識的、分析的な処理です。L2 の推論にも、これと同じ二重構造があります。

過去に繰り返し成功したパターンに基づく推論は、System 1 に相当する「定型推論」として高速に処理されます。一方、前例のない状況や複数領域を横断する複雑な判断は、System 2 に相当する「創発的推論」として、世界モデルの深い探索に基づいて行われます。

また、このレイヤーは組織論者 James Thompson（1967）が分類した3種類の**相互依存性**のうち、最も管理が困難な「相互的依存性」（reciprocal interdependence）に正面から取り組みます。

- **ギャップ分析**: 現在の世界モデルと目的の間の乖離を特定します
- **トレードオフ評価**: 選択肢間のトレードオフを多軸で評価します
- **領域横断構成**: CS・プロダクト・マーケティング・価格など、複数領域を横断する統合的な行動計画を構成します

例えば「顧客維持率 88% → 95%」という目標に対して、CS 部門だけでなく、プロダクトの UX 改善、成功事例コンテンツ、長期契約インセンティブを同時に・整合的に計画し、施策間の相互作用まで考慮します。これが、部門ごとに独立に施策を立案する従来の組織との違いです。各部門が個別に最適化しても全体最適にならないのは、まさに Thompson が指摘した相互依存性の帰結です。

### L3: Execution（実行） — 行動計画 → アウトプット

Execution は、行動計画を実際に実行し、現実世界に成果物を送り出すレイヤーです。メール、レポート、コード、データ操作など多様なアウトプットを生成し、品質チェックを経てデリバリーします。

- **アーティファクト生成**: コミュニケーション、レポート、コード、データ操作の 4 カテゴリの成果物を生成します
- **4段階品質保証**: 基準適合 → 論理整合性 → 安全性 → 最適化の順に検証し、すべてを通過したものだけが Governance Gate に到達します
- **デリバリー**: 適切な受信者に、適切なタイミングで、適切なチャネルで届けます

### L4: Reflection（内省） — アウトプット → 学習 → 全層更新

Reflection は、アウトプットが現実に与えた影響を観測し、システム全体を改善する内省のレイヤーです。単なるフィードバックループではなく、全層を対象とした自己参照的な学習です。

教育学者 Donald Schön（1983）は**省察的実践**（Reflective Practice）の概念を通じて、優れた専門家の学習プロセスを分析しました。彼らは「行為の中の省察」（reflection-in-action）によって、行動しながら同時に自分の行動を振り返り、修正する。L4 はこのプロセスをシステムレベルで実装したものです。

さらに、Argyris（1977）が提唱した**ダブルループ学習** — 行動の結果だけでなく、行動の前提となる価値観や枠組み自体を更新する学習 — が、L4 の本質的な特徴です。L3（実行）の出力品質を改善するだけの「シングルループ」ではなく、L0（知覚すべき対象）や L1（世界の理解の仕方）まで遡って更新するのが「ダブルループ」です。

Flavell（1979）の**メタ認知**（Metacognition）理論 — 自分自身の認知プロセスを監視し制御する能力 — も、L4 の設計に深く関わっています。L4 は「何を考えたか」だけでなく「どのように考えたか」を対象とする、組織のメタ認知機能です。

- **→ Perception**: 接続すべきデータソースの見落としはなかったかを問います
- **→ Understanding**: 世界モデルの因果構造は正しかったかを検証します
- **→ Reasoning**: 判断の前提と行動の構成が最適だったかを振り返ります
- **→ Execution**: 出力の品質は十分だったかを評価します

特に重要なのは、**人間の修正行為そのものが最も価値の高い学習データとなる** ことです。Polanyi（1966）は**暗黙知**（Tacit Knowledge）— 「我々は語ることができるより多くのことを知っている」 — という概念を定式化しました。AIの提案を人間が修正するとき、その修正には言語化されていない組織固有の判断基準が含まれています。提案を修正された場合は暗黙の品質基準を、却下された場合は「何を許容しないか」の境界を学習します。この蓄積が、組織固有の価値観を内面化し、自律性を高めていくプロセスを駆動します。

## 4つの横断的要素

レイヤーがパイプラインの「縦の流れ」だとすれば、横断的要素は「横の力」です。

| 要素 | 役割 | 作用先 |
|---|---|---|
| **Purpose（目的）** | 人間が組織の意志（Why / Where / How / Boundary）を注入する界面 | 主に L2 Reasoning に作用し、行動計画の方向づけを行う |
| **Governance（統治）** | 重要度・不可逆性が閾値を超える判断を人間がゲートする仕組み | L2 と L3 に Gate を設置。信頼スコアに応じて自律範囲が動的に広がる |
| **Memory（記憶）** | 全層にまたがる永続状態のインフラ。長期記憶・作業記憶・評価記憶の3構造 | 全レイヤーが読み書きし、組織の経験を蓄積する |
| **Orchestration（統率）** | 層間の実行フローを制御する指揮官。並列実行、ループバック、スキップ、条件分岐 | 「いつ・どの順序で・どのリソースで」実行するかを決定する |

### Purpose — 意志の注入

Purpose は「指示（Instruction）」ではなく「意志（Intent）」を受け取ります。人間は「メールを送れ」とは指示しません。「顧客の成功を最も重視したい」と意志を伝え、そこから具体的な行動計画が導出されます。

この設計は、Elinor Ostrom（1990）が「コモンズの統治」で示した**立憲的統治**（Constitutional Governance）の原理と対応します。共有資源は、中央からの直接的な命令ではなく、ルールと原則による統治によって最も持続的に管理される。Purpose は、組織という共有資源を統治する「憲法」です。

### Governance — 信頼に基づく段階的委譲

Governance は「承認フロー」ではなく「立憲的統治」です。Davis, Schoorman & Donaldson（1997）の**スチュワードシップ理論**が示す通り、組織のパフォーマンスを最大化するのは、監視ではなく信頼に基づく権限委譲です。信頼スコアが蓄積されるにつれ、システムが自律的に実行できる範囲が段階的に広がります。

Hersey & Blanchard（1977）の**状況適応型リーダーシップ**理論が示す通り、委譲の範囲は相手の成熟度に応じて段階的に広がるべきものです。ただし信頼は領域別に蓄積されるため、CS 領域で Highly Trusted でも、財務領域では Cold Start の可能性があります。

### Memory — 3構造の記憶システム

Memory の3構造 — 長期記憶・作業記憶・評価記憶 — は、Laird, Lebiere & Rosenbloom（2017）の**心の標準モデル**が報告した認知アーキテクチャの記憶構造と対応しています。彼らの分析によれば、3つの独立した認知アーキテクチャがいずれも宣言的長期記憶、バッファ型作業記憶、手続き記憶という共通の構造に到達しました。

組織の記憶もまた、同じ構造的要請に応えています。顧客プロファイルや成功パターンは長期記憶（年〜永続）に、進行中タスクの文脈は作業記憶（秒〜日）に、実行結果の評価は評価記憶（週〜月）に格納されます。

### Orchestration — フロー制御の指揮官

Orchestration は層間の実行フローを制御するメカニズムです。Norbert Wiener（1948）が**サイバネティクス**で定式化した制御理論の原理 — フィードバックを用いた目標追従的制御 — がその理論的基盤です。

W. Ross Ashby（1956）の**必要多様性の法則**（Law of Requisite Variety）は、環境の多様性に対応するには制御側にも十分な多様性が必要であることを示しました。Orchestration が並列実行、ループバック、スキップ、条件分岐といった多様な制御パターンを持つのは、組織が直面する多様な状況に対応するためです。

## 双方向フィードバック — 自己進化する構造

このアーキテクチャの最も重要な特徴は、Reflection（L4）からの学習シグナルが全層にフィードバックされる点です。Friston（2010）の自由エネルギー原理が脳に見出した「予測 → 誤差 → モデル更新」のサイクルを、組織というスケールで実装したものです。

```
L0 (Perception)
  │ シグナル
  ▼
L1 (Understanding)
  │ 世界モデル
  ▼
L2 (Reasoning)  ◄── Purpose  ◄── Governance Gate
  │ 行動計画
  ▼
L3 (Execution)  ◄── Governance Gate
  │ アウトプット
  ▼
L4 (Reflection)
  │ 学習シグナル
  ├──→ L0 更新（接続先・シグナルパターン）
  ├──→ L1 更新（世界モデル・因果モデル）
  ├──→ L2 更新（判断基準・戦略パターン）
  ├──→ L3 更新（品質基準・デリバリーパターン）
  └──→ Memory 更新（長期記憶・評価記憶）
```

たとえば、CS outreach メールの開封率が想定の半分だったとします。Reflection はこの結果を分析し、Perception には「この顧客セグメントの Slack 活動を優先的に監視」するよう更新を送り、Reasoning には「チャネル選択の判断基準」を修正するよう更新を送ります。学習が単一のレイヤーに閉じず、複数レイヤーを横断して伝播するのが、このアーキテクチャの自己進化の仕組みです。

生物学者 Maturana & Varela（1980）は、生命システムが自己の構成要素を自ら生産し続ける仕組みを**オートポイエーシス**（Autopoiesis）と名づけました。Neural Organization の自己進化構造は、このオートポイエーシスの組織的実装です。システムは外部からの入力を受けつつも、自身の構造（パイプラインの設定、接続先、判断基準）を自ら更新し続けます。

## ツール接続の哲学 — 7カテゴリの設計

Perception レイヤーが「現実世界への接続」を担うとき、具体的には以下の 7 カテゴリのツール群との接続を意味します。

| カテゴリ | 対象ツール例 | 取得する情報 |
|---|---|---|
| コミュニケーション | Slack / Teams / LINE | 意思決定の記録、感情の温度感 |
| 顧客関係 | Salesforce / HubSpot / Zendesk | 商談、顧客満足度、解約リスク |
| プロダクト開発 | GitHub / Jira / Linear | PR、スプリント進捗、コード品質 |
| マーケティング | GA / Mixpanel / CMS | PV、CVR、コンテンツ効果 |
| 人事・組織 | HRMOS / SmartHR | エンゲージメント、スキル |
| ファイナンス | freee / Stripe | MRR、キャッシュフロー |
| 運用・インフラ | Datadog / CloudWatch | 稼働状態、インシデント |

ここで重要なのは、**特定のツールに依存しない** 設計思想です。この設計パターンは、Smith（1980）の**Contract Net Protocol** — 分散システムにおいてタスクを能力要件として公告し、能力を持つノードが動的に入札するプロトコル — と同じ哲学に基づいています。Salesforce でも HubSpot でも、Neural Organization にとっては「顧客関係」カテゴリの情報を取得するための接続先に過ぎません。すべての情報は `Customer`、`Deal`、`Product` といった共通エンティティに正規化されます。ツールは入れ替わりますが、組織が知覚すべき「現実の次元」 — 状態・活動・関係・変化 — は不変です。

Ashby（1956）の**必要多様性の法則**が示す通り、組織が対応すべき環境の多様性に見合うだけの知覚チャネルが必要です。7カテゴリは、現代のビジネス組織が知覚すべき環境の最小限の多様性をカバーする設計です。

接続は段階的に進みます。最初は Slack、CRM、GitHub の3つを接続するだけで、組織の「いま何が起きているか」の 60〜70% が知覚可能になります。そこから Reflection が検出した盲点に基づいて、システム自身が「次に何に接続すべきか」を判断し、接続先を広げていきます。

## 透明な状態管理 — すべてにパスがある

Neural Organization では、組織のあらゆる状態を一意のパスでアドレッシングできます。ファイルシステムのように、すべての情報が名前を持ち、誰でも参照できる仕組みです。

```
/entities/customers/acme-corp          # 顧客エンティティ
/entities/deals/acme-expansion-2024    # 商談エンティティ
/relations/customers/acme-corp/deals   # 顧客→商談の関係
/metrics/retention-rate/latest         # 最新の維持率
/sessions/{session-id}/context         # 進行中タスクの文脈
/patterns/success/early-onboarding     # 成功パターン
```

この設計には、いくつかの重要な判断があります。

**フラットな 2 階層構造**: パスの深さは最大 2 階層（`/{名前空間}/{エンティティ種別}/{識別子}`）に制限しています。深いネストはエージェントの認知負荷を高め、パフォーマンスも低下させるためです。エンティティ間の関係性は `/relations/` 配下のグラフクエリで辿ります。

**Memory の 3 構造との対応**: 名前空間がそのまま記憶構造にマッピングされます。`/entities/` や `/patterns/` は長期記憶（年〜永続）、`/sessions/` は作業記憶（秒〜日）、`/metrics/` は評価記憶（週〜月）です。主要なアクセスパスで記憶構造を意識する必要はありませんが、時間特性が異なるデータは明示的に分離されています。

**進化への対応**: パス構造の変更にはエイリアスレジストリ方式を採用しています。旧パスは新パスに自動リダイレクトされ、12ヶ月の猶予期間が設けられます。パス構造自体も `/config/path-schema` にデータとして格納されるため、Reflection による自己改善の対象になります。Event Sourcing パターンや GitOps の実践が示す通り、構成と状態のデータ化は、システムの自己参照的な進化の前提条件です。

この「すべての状態にアドレスがある」仕組みが、Radical Transparency（徹底的な透明性）原則の技術的な基盤です。エージェントも人間も、同じパスを使って組織の状態を参照できます。ブラックボックスはありません。

## まとめ — 5層が「考える組織」を実現する

Neural Organization の 5 層アーキテクチャは、組織の知的活動を「知覚 → 理解 → 推論 → 実行 → 内省」の変換パイプラインとして構造化します。この構造は、Felleman & Van Essen が脳に見出した分散階層処理、Boyd が戦場に見出した OODA ループ、Laird らが認知アーキテクチャに見出した心の標準モデルと、同じ設計原理を共有しています。

各レイヤーは明確な入力と出力を持ち、横断的要素がパイプライン全体に方向性（Purpose）、統治（Governance）、記憶（Memory）、制御（Orchestration）を与えます。そして Reflection が生み出す学習シグナルが全層にフィードバックされることで、システムは使うほどに組織の文脈を深く理解し、自律的に進化していきます。

次回は、この 5 層アーキテクチャの中核を流れる「人間-AI 界面」のインタラクション設計について掘り下げます。人間はこのシステムとどう対話し、どのように組織の意志を注入するのか。Intent over Instruction、Governance not Control といった設計原理が、具体的なインターフェースとしてどう実現されるのかを解説します。
