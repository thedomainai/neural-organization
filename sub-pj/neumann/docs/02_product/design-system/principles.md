# Design Principles

neumannの設計原則。すべてのデザイン判断の基盤となる。

## 5つの原則

### 1. Clarity First（明瞭性優先）

> 「迷わせない、考えさせない」

- 情報階層を明確にする
- 1画面1目的を徹底する
- 曖昧な表現・アイコンを避ける
- ラベルは具体的に、動詞で始める

**Good**: 「レポートを提出」「3件の問題を修正」
**Bad**: 「送信」「確認」「処理」

---

### 2. Respectful Density（敬意ある密度）

> 「経営者の時間を尊重する」

- 5秒ルール: 主要な情報は5秒以内に把握可能
- 8-12 KPIs: 1画面に表示するKPIは最大12個
- Progressive Disclosure: 詳細は必要なときだけ表示
- スクロールを最小化、フォールド上に重要情報

**情報密度の基準:**
```
ダッシュボード: 高密度（データ中心）
エディター:     中密度（作業中心）
設定:          低密度（操作中心）
```

---

### 3. Quiet Confidence（静かな自信）

> 「過剰な装飾は不信を招く」

- 装飾よりコンテンツを優先
- アニメーションは機能的な場合のみ
- 色は意味を持つ場合のみ使用
- 余白は贅沢ではなく、機能である

**色の使用基準:**
- グレースケールをベースに
- 色はステータス・アクションの識別にのみ使用
- ブランドカラー（Navy）はヘッダー・重要アクセントに限定

---

### 4. Trustworthy Data（信頼できるデータ）

> 「数字は正確に、文脈と共に」

- データの出典・更新時刻を明示
- 比較対象（前週比、目標比）を常に提供
- 異常値は自動的にハイライト
- グラフは歪曲しない（Y軸は0から）

**データ表示の原則:**
```
数値:     右揃え、等幅フォント
単位:     数値の直後、小さめサイズ
変化:     矢印 + 色（緑↑ 赤↓）
更新時刻: フッター、グレー、小さめ
```

---

### 5. Accessible Authority（アクセシブルな権威）

> 「プロフェッショナルだが、威圧しない」

- 専門用語は避け、平易な言葉を使う
- エラーは責めず、解決策を示す
- 操作の結果を即座にフィードバック
- キーボード操作を完全サポート

**トーン:**
```
Good: 「3件の改善点が見つかりました」
Bad:  「3件のエラーが検出されました」

Good: 「保存しました」
Bad:  「保存処理が正常に完了しました」
```

---

## 原則の適用チェックリスト

新しいUIを設計・レビューする際に確認：

### Clarity First
- [ ] 1画面の目的は明確か？
- [ ] ラベルは具体的で動詞から始まっているか？
- [ ] 初見でも操作方法が分かるか？

### Respectful Density
- [ ] 5秒で主要情報を把握できるか？
- [ ] KPIは12個以下か？
- [ ] 詳細は適切に隠されているか？

### Quiet Confidence
- [ ] 装飾的な要素は本当に必要か？
- [ ] 色は意味を持っているか？
- [ ] 余白は十分に確保されているか？

### Trustworthy Data
- [ ] データの出典・更新時刻は明示されているか？
- [ ] 比較対象は提供されているか？
- [ ] グラフは正確か（Y軸0から）？

### Accessible Authority
- [ ] 専門用語は避けられているか？
- [ ] エラーメッセージは解決策を含んでいるか？
- [ ] キーボード操作は可能か？

---

## アンチパターン

避けるべきデザインパターン：

| アンチパターン | 問題 | 代替案 |
|---------------|------|--------|
| 虹色グラデーション | 安っぽさ、視認性低下 | 単色 or 2色グラデ |
| 角丸過多 | 子供っぽさ | 4-8pxに統一 |
| ドロップシャドウ多用 | 重たい印象 | 最小限、または border |
| アイコンのみボタン | 意味不明 | ラベル併記 |
| 赤以外の警告色 | 緊急性が伝わらない | 赤を使う |
| 自動カルーセル | 操作感の喪失 | 手動 or 廃止 |
