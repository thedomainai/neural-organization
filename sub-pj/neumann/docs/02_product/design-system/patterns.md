# UI Patterns

neumannの共通UIパターン。

## Page Patterns

### Dashboard Layout

経営者が最初に見る画面。5秒で状況把握可能な設計。

```
┌─────────────────────────────────────────────────────────────┐
│ [Header] neumann          [通知] [ユーザー]                 │
├─────────┬───────────────────────────────────────────────────┤
│         │                                                   │
│ [Nav]   │  [Greeting] おはようございます、山田さん          │
│ ダッシュ │  [Date] 2026年2月1日 (土)                        │
│ レポート │                                                   │
│ 設定    │  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ │
│         │  │ KPI 1   │ │ KPI 2   │ │ KPI 3   │ │ KPI 4   │ │
│         │  │ 87点    │ │ 12件    │ │ 3件     │ │ 95%     │ │
│         │  └─────────┘ └─────────┘ └─────────┘ └─────────┘ │
│         │                                                   │
│         │  ┌─────────────────────────────────────────────┐ │
│         │  │ [Chart] 品質スコア推移                       │ │
│         │  │                                             │ │
│         │  └─────────────────────────────────────────────┘ │
│         │                                                   │
│         │  ┌─────────────────────┐ ┌─────────────────────┐ │
│         │  │ [List] 要対応       │ │ [List] 最近の活動  │ │
│         │  │ レポート            │ │                     │ │
│         │  └─────────────────────┘ └─────────────────────┘ │
│         │                                                   │
└─────────┴───────────────────────────────────────────────────┘
```

**KPIカード優先順位:**
1. 品質スコア（メイン指標）
2. 要対応レポート数
3. 今週の改善数
4. 提出率

### Editor Layout

レポート編集画面。左に編集、右に検出結果。

```
┌─────────────────────────────────────────────────────────────┐
│ [Header] ← 戻る  レポートタイトル     [保存] [提出]        │
├─────────────────────────────────────────────────────────────┤
│ [Toolbar] B I U | H1 H2 | • 1. | — |                        │
├────────────────────────────────┬────────────────────────────┤
│                                │                            │
│  [Editor]                      │  [Audit Panel]             │
│                                │                            │
│  テキストエリア                │  品質スコア: 87点          │
│                                │  ─────────────────         │
│  ここにレポート内容を          │  ⚠ 3件の改善点             │
│  入力します...                 │                            │
│                                │  □ 定量性不足 (L12)        │
│  [ハイライトされた             │  □ 行動不明確 (L24)        │
│   問題箇所]                    │  □ 深掘り不足 (L31)        │
│                                │                            │
│                                │  [改善提案を見る]          │
│                                │                            │
└────────────────────────────────┴────────────────────────────┘
```

**パネル比率:** 60% : 40% (リサイズ可能)

## Interaction Patterns

### Progressive Disclosure

詳細は必要なときだけ表示。

```
[要約表示]
┌─────────────────────────────────────┐
│ 品質スコア: 87点  [詳細を見る ▼]   │
└─────────────────────────────────────┘
           ↓ クリック
[展開表示]
┌─────────────────────────────────────┐
│ 品質スコア: 87点  [閉じる ▲]       │
├─────────────────────────────────────┤
│ 内訳:                               │
│ ・定量性:     90点                  │
│ ・明確性:     85点                  │
│ ・網羅性:     86点                  │
└─────────────────────────────────────┘
```

---

### Inline Edit

その場で編集。モーダル不要。

```
[表示モード]
┌─────────────────────────────────────┐
│ レポートタイトル        [✏️ 編集]   │
│ 2026年1月第4週 営業部               │
└─────────────────────────────────────┘
           ↓ 編集クリック
[編集モード]
┌─────────────────────────────────────┐
│ レポートタイトル                    │
│ ┌─────────────────────────────────┐ │
│ │ 2026年1月第4週 営業部           │ │
│ └─────────────────────────────────┘ │
│                    [キャンセル] [保存] │
└─────────────────────────────────────┘
```

---

### Confirmation Dialog

破壊的操作の前に確認。

```
┌─────────────────────────────────────┐
│ ⚠️ レポートを削除しますか？         │
├─────────────────────────────────────┤
│                                     │
│ 「2026年1月第4週 営業部」を         │
│ 削除します。この操作は取り消せません。│
│                                     │
├─────────────────────────────────────┤
│           [キャンセル] [削除する]   │
└─────────────────────────────────────┘
```

**ルール:**
- 削除ボタンは右側（destructive）
- キャンセルが逃げやすい位置（左側）
- 対象を明示（タイトル名など）

---

### Empty State

データがない場合の表示。

```
┌─────────────────────────────────────┐
│                                     │
│         📄                          │
│                                     │
│   レポートがありません              │
│                                     │
│   最初のレポートを作成して          │
│   品質改善を始めましょう            │
│                                     │
│        [+ レポートを作成]           │
│                                     │
└─────────────────────────────────────┘
```

**要素:**
- アイコン（大きめ、グレー）
- タイトル（何がないか）
- 説明（どうすればいいか）
- アクションボタン

---

### Loading State

読み込み中の表示。

```
[スケルトン]
┌─────────────────────────────────────┐
│ ████████████████                    │
│ ██████████                          │
│                                     │
│ ████████████████████████████        │
│ ████████████████████                │
└─────────────────────────────────────┘

[スピナー（ボタン）]
┌─────────────────────────────────────┐
│        [⟳ 処理中...]                │
└─────────────────────────────────────┘
```

**ルール:**
- 全画面ロードはスケルトン
- ボタンアクションはスピナー
- 200ms以下は表示しない

---

### Error State

エラー時の表示。

```
[インラインエラー（フォーム）]
┌─────────────────────────────────────┐
│ レポートタイトル                    │
│ ┌─────────────────────────────────┐ │
│ │                                 │ │
│ └─────────────────────────────────┘ │
│ ⚠️ タイトルは必須です               │
└─────────────────────────────────────┘

[ページエラー]
┌─────────────────────────────────────┐
│                                     │
│         ⚠️                          │
│                                     │
│   読み込みに失敗しました            │
│                                     │
│   ネットワーク接続を確認して        │
│   再度お試しください                │
│                                     │
│        [再読み込み]                 │
│                                     │
└─────────────────────────────────────┘
```

**ルール:**
- エラーは赤（error-600）
- 解決策を必ず提示
- 再試行ボタンを提供

## Form Patterns

### Form Layout

```
┌─────────────────────────────────────┐
│ セクションタイトル                  │
├─────────────────────────────────────┤
│ ラベル *                            │
│ ┌─────────────────────────────────┐ │
│ │ 入力値                          │ │
│ └─────────────────────────────────┘ │
│ ヘルプテキスト                      │
│                                     │
│ ラベル                              │
│ ┌─────────────────────────────────┐ │
│ │ 入力値                          │ │
│ └─────────────────────────────────┘ │
│                                     │
├─────────────────────────────────────┤
│              [キャンセル] [保存]    │
└─────────────────────────────────────┘
```

**ルール:**
- ラベルは入力の上
- 必須は * マーク
- ヘルプテキストは入力の下
- アクションボタンは右下
