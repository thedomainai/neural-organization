# Value Proposition

提供価値を定義する。

## 一言で言うと

**「定例資料の曖昧さをゼロにし、経営会議の生産性を最大化するAI」**

週次定例・MBRで「なぜ？」「具体的には？」「誰がいつまで？」と指摘される前に、AIが事前に曖昧さを検知・排除。手戻りゼロの解像度の高い報告を実現する。

---

## バリュープロポジションキャンバス

### 顧客セグメント①: CEO（購入決定者）

**Jobs（顧客の仕事）**
- 組織全体の状況を把握し、意思決定を行う
- 定例会議で各部門の進捗・課題を確認する
- 問題があれば早期に介入し、軌道修正する

**Pains（課題・不満）**
- 定例資料が曖昧で、毎回「なぜ？」「具体的には？」と深掘りが必要
- 報告の品質がマネージャーによってバラバラ
- 本質的な議論ではなく、情報の確認に時間を浪費
- 自分が指摘し続けることで「嫌われ役」になる疲弊
- Slack/資料を自分で確認しないと本当のことがわからない（Black Box Anxiety）

**Gains（望む成果）**
- 定例開始時点で、解像度の高い報告が揃っている状態
- 会議時間を情報確認ではなく意思決定に使える
- 自分が指摘しなくても、組織に規律が維持される（Trusted Bad Cop）
- 組織の隅々まで把握できている全能感（Observability）

---

### 顧客セグメント②: 現場マネージャー（利用者）

**Jobs（顧客の仕事）**
- 週次定例・MBRで自部門の進捗・課題を報告する
- Google Slides / Docs / Notionで報告資料を作成する
- CEOや上長からの質問に答える

**Pains（課題・不満）**
- 何をどこまで書けば「十分」かわからない
- 定例で指摘されて手戻りが発生、時間を浪費
- 深掘りが足りないと言われるが、どう深掘りすればいいかわからない
- 定例前に「これで大丈夫か」と不安になる

**Gains（望む成果）**
- 事前に何を書けばいいか明確にガイドされる
- 定例で指摘されず、スムーズに報告が終わる
- 報告スキルが向上し、経営視点が身につく
- 「よくできた報告だ」とCEOに評価される

## バリュープロポジション

### Products & Services（製品・サービス）

| 機能 | 説明 |
|------|------|
| **曖昧さ検知エンジン** | 4パターン（深掘り/網羅性/定量性/行動）の曖昧さを自動検知 |
| **質問/指示生成** | 検知した曖昧さに対して具体的な質問・修正指示を提示 |
| **対話型作成支援** | チャットで回答するだけで資料が改善される |
| **タスク管理** | 未解決の曖昧さをタスク化、通知でリマインド |
| **品質スコア** | 資料の曖昧さ排除度合いを可視化 |

### Pain Relievers（課題解決）

| Pain | Reliever |
|------|----------|
| 毎回「なぜ？」と深掘りが必要 | 定例前にAIが深掘り完了、CEOの認知負荷ゼロ |
| 品質がマネージャーによりバラバラ | 一貫した基準で全員に同じ品質を要求 |
| 情報確認に会議時間を浪費 | 定例開始時点で情報は揃っている |
| 嫌われ役になる疲弊 | AIがBad Cop、CEOは意思決定に集中 |
| 何を書けばいいかわからない | AIが具体的に質問・指示、迷わない |
| 定例で指摘されて手戻り | 事前に曖昧さ排除、手戻りゼロ |

### Gain Creators（価値創造）

| Gain | Creator |
|------|---------|
| 全能感（Observability） | 全資料の品質・進捗をダッシュボードで一覧 |
| Trusted Bad Cop | AIが自律的に規律を維持、CEO不在でも機能 |
| 報告スキル向上 | 繰り返し指摘を受けることで経営視点が身につく |
| 評価される報告 | 品質スコアが高い＝CEOに評価される報告 |
