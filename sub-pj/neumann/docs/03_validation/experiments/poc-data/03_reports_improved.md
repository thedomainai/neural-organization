## 曖昧性排除のチェックリスト

レポート作成時に確認すべき項目：

### 1. 事実と解釈の分離 `[FI]`

- [ ] 「良好」「順調」「期待できる」などの形容詞は、具体的な事実に置き換えたか
- [ ] 他者の発言を引用する場合、発言者と内容を明記したか
- [ ] 「〜と思う」「〜かもしれない」は、根拠となるデータを添えたか

### 2. 定量化 `[LQ]`

- [ ] すべての指標に数値（実績値、目標値、達成率）を記載したか
- [ ] 「増加」「減少」には先週比/先月比の数値を添えたか
- [ ] 「やや」「少し」「多い」などの程度を表す言葉は数値に置き換えたか

### 3. 原因分析の深さ `[SA]`

- [ ] 問題が発生した場合、「なぜ」を最低2回深掘りしたか
- [ ] 原因を構成要素に分解（例: 要因A 30%、要因B 50%、要因C 20%）したか
- [ ] 対策が原因に対応しているか確認したか

### 4. アクションの明確さ `[UA]`

- [ ] すべてのアクションにWho（担当）を明記したか
- [ ] すべてのアクションにWhen（期限、具体的な日付）を明記したか
- [ ] すべてのアクションにWhat（完了基準）を明記したか

### 5. 網羅性 `[MC]`

- [ ] 報告対象すべてについて言及したか（例: 赤アカウント5社すべて）
- [ ] 例外や未対応の項目を明示したか
- [ ] 「〜など」で省略していないか

## PoC検証での使用方法

1. `01_reports_raw.md` をLLMに入力
2. LLMに曖昧性の検出を依頼
3. LLMの出力と `02_reports_annotated.md` を比較
4. Precision/Recall を算出

**期待する検出精度**:
- Precision: 80%以上（誤検出が少ない）
- Recall: 70%以上（見逃しが少ない）

**許容する誤差**:
- Severity（重要度）の1段階ずれは許容
- Pattern（パターン）の軽微な混同は許容（例: SA と LQ の境界）
