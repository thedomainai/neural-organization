# Agentic AI Management 概要

> 旧称: neumann。経営管理サイクルを AI が並走し、計画と現実の乖離を「必然的に」縮小させるプロダクト。

## WHY: なぜこのプロダクトが必要か

### Vision

**「経営管理の再現性の民主化」**

KPI 設計から問題の特定、仮説検証に至るサイクル（KPI 設計 - 問題箇所の特定 - 課題の特定 - 打ち手仮説 - 検証）が正しく回れば、あらゆる連続的な経営課題は解決される。未達の要因を原子レベル（A-1, A-2...）まで分解し、曖昧さを完全に排除することで、誰もが再現性を持って事業を成長させられる世界を実現する。

### Mission

**「ビジョンと実装の間にある『曖昧さ』を排除し、成果を『必然』にする」**

スタートアップ経営における不確実性を管理可能な変数へと変換する。事実と解釈を厳格に分離し、論理に基づいた予実管理と自律的な AI の支援を通じて、組織をゴールへと力強くナビゲートする。

### Core Purpose: なぜ Agentic AI なのか

人間を超える知能の最大活用と、事業経営に不要な人間的なノイズの排除。

感情・政治・忖度といった「ノイズとなる人間的なコンテキスト」を弾き、純粋なデータと論理、そして定義されたコンテキストのみに基づいて駆動する。これにより、人間には維持困難な高い解像度と頻度で問題解決のサイクルを回し続ける。

### 提供価値

| 対象 | 価値 |
|---|---|
| **CEO** | 曖昧さ・不明瞭さから解放。全てが手に取るようにわかる全能感と、権限移譲による本来の力の解放 |
| **Business** | 論理的矛盾や曖昧さの排除による、スループットの最大化 |
| **Operation** | 結果としての予実管理ではなく、根本的な「問題解決フレームワーク」の確立 |

## WHO: ステークホルダー

### プライマリペルソナ: "The Lonely Visionary" (孤高の論理的 CEO)

- **企業規模**: シリーズ A-B（30-100 名）から時価総額 1 兆円（数千-数万名）
- **業種**: SaaS / Tech / 高い成長率を求められる業態
- **年齢層**: 20 代後半 - 50 代
- **テクノロジーリテラシー**: 極めて高い。人間よりもソフトウェアを信頼する傾向
- **意思決定スタイル**: Logic-First & Fast。感情論や根拠のない楽観を嫌う
- **AI からの指摘への態度**: 自分自身への厳しい指摘も受け入れる。「正しければ従う」

**スケール別の課題深刻度**:

| 企業規模 | 課題の深刻度 | Black Box Anxiety | 代替手段のコスト |
|---|---|---|---|
| シリーズ A-B | 高 | 中（まだ自分で見える） | COO 1500 万- |
| 時価総額 100 億- | 極めて高 | 高（報告が多層化） | COO 3000 万- |
| 時価総額 1000 億- | 最大 | 極めて高（情報が歪曲） | COO 5000 万-1 億 |

**主要ペイン**:
- Black Box Anxiety: 事業の細部で何が起きているか、現場からの報告がフィルタリングされており本当のことがわからない恐怖。組織が大きくなるほど深刻化
- Cognitive Overload: メンバーの報告内容の裏取りや論理的整合性チェックに膨大な認知リソースを消耗
- Execution Distrust: 「任せる」と言いつつ、結局自分で確認しないと品質が担保できない徒労感

**求める成果**:
- Observability（全能感）: 組織の隅々まで数値とロジックで把握できている状態
- Trusted Bad Cop: 自分が嫌われ役にならずとも規律が維持される安心感
- Zero Latency: 意思決定に必要なファクトが質問前に揃っている状態

**現在の代替手段**:
- 自分自身: 睡眠時間を削って Slack を全部読む
- Human COO: 採用コストが高く（年収 1500 万-1 億）、カルチャーフィットのリスクが高い
- Chief of Staff: 気が利くが、経営判断や論理的な指摘まではできない
- 経営企画部門: 大企業では存在するが、CEO への忖度が働き Bad Cop として機能しない

### セカンダリペルソナ: "The Operation Managers" (現場マネージャー)

本プロダクトにおいて、彼らは能動的なユーザーではなく、AI による「監査・支援対象」である。

- **役職**: Sales Manager, CS Lead, PdM など
- **特徴**: プレイングマネージャーとして多忙
- **AI との関係**: AI からの冷徹な指摘（Fact-Checking）を受ける対象。当初は「監視役」として警戒するが、AI との対話を通じて論理的な課題解決（分解・構造化）を学習し、経営幹部候補へと育成される
- **注記**: 現場の反発への対応はオペレーション（導入支援・チェンジマネジメント）で行い、プロダクト機能としては考慮しない

### アンチペルソナ

- "Vibe" CEO: ロジックよりも熱意・調和を最優先する経営者
- Micro-Manager: 自分で細かく指示すること自体に快感を覚えるタイプ
- Defensive CEO: 指摘されると感情的に反発するタイプ
- AI 介入拒否組織: ガバナンス上 AI の自律介入を許容できない組織

## WHAT: 機能と特徴

### 経営管理サイクル最適化

v2 ではプロダクトを「週次報告の曖昧さ検知ツール」から「経営管理サイクル全体を AI が並走するプロダクト」へと再構築する。

```
┌──────────────────────────────────────────────────────────┐
│                 経営管理サイクル                            │
│                                                          │
│    ┌──────────┐     ┌──────────┐     ┌──────────────┐   │
│    │ 予実管理  │────→│ 差異分析  │────→│ ネクストアクション│   │
│    │ (Plan    │     │ (Analyze │     │ 定義 (Act)    │   │
│    │  vs Act) │     │  Gap)    │     │               │   │
│    └────▲─────┘     └──────────┘     └──────┬────────┘   │
│         │                                     │          │
│         └─────────────────────────────────────┘          │
│              結果の反映 (Reflect & Update)                 │
└──────────────────────────────────────────────────────────┘
```

| 工程 | 内容 |
|---|---|
| **予実管理** | KPI 定義、自動取得、達成率計算、異常検知 |
| **差異分析** | 定量差異の自動検知 + 報告品質担保 + 因果分析支援 |
| **ネクストアクション** | アクション案の自動生成 + Who/What/When の明確化 |
| **結果の反映** | アクション効果の追跡 + 予測モデルの更新 |

### 曖昧さ検知エンジン（v1 コア機能）

Zero Ambiguity の原則に基づき、5 つのパターンを検知する。

| パターン | 説明 |
|---|---|
| 事実/解釈混在 | Fact と Interpretation の混同を検知 |
| 定量化不足 | 数値がない、定性的すぎる報告 |
| アクション不明確 | Who/What/When が不明 |
| 浅い分析 | 原子レベルまで分解されていない |
| カバレッジ不足 | MECE でない、抜け漏れ |

### コア原則

1. **Zero Ambiguity**: 曖昧さをバグとして扱う
2. **Fact/Interpretation Separation**: 事実と解釈の厳格な分離
3. **Recursive Resolution**: 問題解決の無限ループを止めない
4. **Risk/Outcome Isolation**: リスクと成果を独立した変数として扱う
5. **Autonomous Intervention**: 聞かれる前に介入するパートナー

## Neural Organization における位置づけ

Agentic AI Management は Neural Organization エコシステムの中核プロジェクトの一つである。経営管理という「組織が目的を達成しているかを継続的に検証し修正するプロセス」を AI 化する。

**他プロジェクトとの関係**:
- **ai-executive-dashboard**: 市場トレンドデータの受信元。差異分解の市場要因推定に活用
- **agentic-ai-hr**: 等級定義の読み出し元。報告品質基準の動的調整に活用。また報告品質スコアの書き出し先

## 命名由来

旧名 neumann は、ジョン・フォン・ノイマン（John von Neumann, 1903-1957）に由来する。ゲーム理論の創始者であり計算機科学の父でもある彼は、経営管理と AI の交差点に立つ本プロジェクトの思想的な象徴であった。Neural Organization のサブプロジェクトとしての位置づけを明確にするため、agentic-ai-management に改称された。
