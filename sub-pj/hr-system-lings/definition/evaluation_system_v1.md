# 人事制度：評価制度 (Evaluation System)

## 1. 評価コンセプト：卒業要件方式
本制度の評価は、他者との比較（相対評価）ではなく、現グレードに定められた要件を完全に満たし、次のステージへ進む準備ができているか（絶対評価）を判定する。

## 2. 昇格・昇進の判定ロジック
昇格候補者として「キャリブレーション会議」に上程されるためには、以下の**3つのゲートをすべてクリア**する必要がある。一つでも欠けた場合は、現グレードにステイとなる。

### Step 1：コンピテンシー評価 【卒業要件】
現グレードに定義された3つのコンピテンシー（思考・行動・対人）の発揮度。
- **採点**: 1〜5点の5段階評価
- **昇格要件**: **3領域すべてにおいて「4点（Exceed）」以上**であること。
    - ※一つでも3点（Meet）がある場合は「現グレードにおいてまだ伸び代がある」と判断され、卒業とはみなされない。

### Step 2：成果評価 【チケット】
期間内に合意した目標（MBO/OKR等）に対する達成度。
- **判定**: ⚪︎（達成）・△（概ね達成）・×（未達）
- **昇格要件**: **「⚪︎」以上**であること。

### Step 3：振る舞い（バリュー）評価 【ゲートキーパー】
組織の行動規範（バリュー）の体現度。
- **判定**: ⚪︎（体現）・△（一部課題）・×（乖離）
- **昇格要件**: **「⚪︎」以上**であること。（×は即座に足切りとなる）

---

## 3. 採点基準（コンピテンシー評価）

| 点数 | 判定 | 定義 |
| :--- | :--- | :--- |
| **5** | **Outstanding** | **卓越**：上位グレードの要件すらも一部満たしており、組織に極めて高い付加価値を与えている。 |
| **4** | **Exceed** | **超過（卒業レベル）**：現グレードの期待値を常に上回り、安定して高いレベルの行動を再現できている。 |
| **3** | **Meet** | **充足**：現グレードの期待値を概ね満たしている。プロとして標準的な動きができている。 |
| **2** | **Below** | **不足**：期待値を下回ることがあり、周囲のサポートや指導を頻繁に必要とする。 |
| **1** | **Unsatisfactory**| **未達**：期待値を大きく下回り、改善の兆しが見られない。 |

## 4. 運用サイクル

### 4.1 定常運用（成長モニタリング）
- **月次 1on1**: 
    - 評価者と被評価者による実施。
    - 直近の具体的エピソードに基づき、コンピテンシー発揮状況をリアルタイムにフィードバックする。
- **中間レビュー（3ヶ月に1回）**:
    - 期末評価の「予演」として、現時点での評価（⚪︎△×および点数）をすり合わせる。
    - 昇格に向けて不足している要素を特定し、後半3ヶ月のアクションプランを合意する。

### 4.2 期末評価プロセス（年2回）

#### 評価サイクル
| 期 | 対象期間 | 評価実施時期 |
| :--- | :--- | :--- |
| **上期** | 12月〜5月 | 6月 |
| **下期** | 6月〜11月 | 12月 |

#### 評価プロセス
1. **自己評価・上長評価**: 事実（Fact）に基づき判定を行う。
2. **昇格候補の抽出**: 上記Step 1〜3をすべて満たす者をシステム的に抽出。
3. **キャリブレーション会議**:
    - 経営層・本部長級による審議。
    - 全社視点での評価基準の目合わせを行い、最終的な昇格者を決定する。
4. **フィードバック面談**:
    - 結果の伝達だけでなく、**「次の半年の目標（Next Gradeの要件）」**を言語化し、本人の動機付けを行う。
