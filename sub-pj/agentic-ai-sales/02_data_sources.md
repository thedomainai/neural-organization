# データソース分類 - プリセット vs ユーザー入力

## 概要

プレゼン資料生成に必要なデータを、取得方法別に4つに分類する。

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         データソース分類                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐    │
│  │   PRESET    │  │   MASTER    │  │    AUTO     │  │    USER     │    │
│  │  プリセット  │  │   マスタ    │  │   自動取得   │  │  ユーザー入力 │    │
│  └─────────────┘  └─────────────┘  └─────────────┘  └─────────────┘    │
│        ↓                ↓                ↓                ↓            │
│   初期設定時         継続的更新        リアルタイム      商談ごと         │
│   1回設定           管理画面で         API/Web          都度入力         │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

---

## 1. PRESET - プリセット（初期設定・1回）

> **設定タイミング**: SaaS導入時、初期オンボーディング時
> **更新頻度**: 年1-2回（ブランド変更時など）
> **設定者**: 管理者、マーケティング担当

### 1-1. ブランド設定

```
ブランド設定（PRESET）
├── 企業情報
│   ├── 会社正式名称
│   ├── 会社略称
│   ├── ロゴ（各種バリエーション）
│   ├── コーポレートカラー（HEXコード）
│   └── キャッチコピー/タグライン
│
├── デザインシステム
│   ├── カラーパレット
│   │   ├── プライマリカラー
│   │   ├── セカンダリカラー
│   │   ├── アクセントカラー
│   │   ├── テキストカラー（濃/中/淡）
│   │   └── 背景カラー
│   │
│   ├── タイポグラフィ
│   │   ├── 見出しフォント（日本語/英語）
│   │   ├── 本文フォント（日本語/英語）
│   │   ├── フォントサイズ規定
│   │   └── 行間・字間規定
│   │
│   └── ビジュアルガイドライン
│       ├── 写真のトーン
│       ├── イラストのスタイル
│       ├── アイコンセット
│       └── 使用禁止表現
│
├── スライドテンプレート
│   ├── マスタースライド一式
│   ├── タイトルスライド
│   ├── アジェンダスライド
│   ├── セクション区切り
│   ├── コンテンツ各種
│   └── エンドスライド
│
└── 法的・コンプライアンス設定
    ├── 免責事項テンプレート
    ├── 著作権表記
    └── 必須表示事項
```

### 1-2. 制約条件設定

```
制約条件（PRESET）
├── 禁止表現
│   ├── 絶対表現（「絶対」「必ず」「100%」）
│   ├── 保証表現（「保証」「確実」）
│   ├── 誇大表現リスト
│   ├── 競合誹謗表現
│   └── 業界固有NG表現
│
├── 必須表現
│   ├── 数値の出典明記ルール
│   ├── 免責表示ルール
│   └── 商標表記ルール
│
└── スタイルルール
    ├── 文体（です/ます or だ/である）
    ├── 主語（弊社/当社/私たち）
    ├── 敬語レベル
    └── 専門用語の平易化ルール
```

### 1-3. ストーリーフレームワーク定義

```
ストーリーフレームワーク（PRESET）
├── SPIN
│   ├── Situation → Problem → Implication → Need-payoff
│   └── 推奨シナリオ: 1次商談
│
├── 問題解決型
│   ├── 課題 → 原因 → 解決策 → 効果 → 実装計画
│   └── 推奨シナリオ: 詳細提案
│
├── Before/After
│   ├── 現状 → 理想 → ギャップ → 解決策 → 成果
│   └── 推奨シナリオ: 事例紹介
│
└── カスタムフレームワーク
    └── 企業独自のストーリー構成
```

## 2. MASTER - マスタデータ（継続的更新）

> **設定タイミング**: 継続的（週次/月次）
> **更新頻度**: 新製品、新事例、競合変化時
> **設定者**: 営業企画、マーケティング、プリセールス

### 2-1. 製品情報マスタ

```
製品情報（MASTER）
├── 製品カタログ
│   ├── 製品名
│   ├── 製品概要（1行/3行/詳細）
│   ├── 主要機能リスト
│   ├── 差別化ポイント
│   ├── 対象顧客像
│   └── スクリーンショット
│
├── 価格情報
│   ├── 価格表（プラン別）
│   ├── オプション価格
│   ├── ボリュームディスカウント
│   └── 価格改定履歴
│
├── 技術仕様
│   ├── システム要件
│   ├── API仕様
│   ├── セキュリティ認証
│   └── SLA
│
└── FAQ
    ├── よくある質問と回答
    ├── 想定反論と切り返し
    └── 回答難度
```

### 2-2. 事例マスタ

```
事例情報（MASTER）
├── 導入事例
│   ├── 顧客名（公開可否）
│   ├── 業界
│   ├── 企業規模
│   ├── 導入製品
│   ├── 課題（Before）
│   ├── 解決策
│   ├── 成果（After）
│   ├── 定量効果（数値）
│   ├── 顧客の声（引用）
│   └── ロゴ使用可否
│
└── 事例検索条件
    ├── 業界タグ
    ├── 課題タグ
    ├── 規模タグ
    └── 製品タグ
```

### 2-3. 競合情報マスタ

```
競合情報（MASTER）
├── 競合プロファイル
│   ├── 競合企業名
│   ├── 競合製品名
│   ├── ポジショニング
│   ├── 強み
│   ├── 弱み
│   └── 価格帯
│
├── 比較表
│   ├── 機能比較表
│   ├── 価格比較表
│   └── 導入実績比較
│
└── 競合対策
    ├── 差別化メッセージ
    ├── 想定反論と切り返し
    └── 勝ちパターン
```

### 2-4. 勝ち/負けパターン

```
パターン学習（MASTER）
├── 勝ちパターン
│   ├── 成功案件の提案資料
│   ├── 成功要因タグ
│   │   ├── 刺さったメッセージ
│   │   ├── 効果的だった構成
│   │   └── 決め手になったスライド
│   ├── 顧客属性との相関
│   └── ベストプラクティス
│
├── 負けパターン
│   ├── 失注案件の提案資料
│   ├── 失敗要因タグ
│   └── アンチパターン
│
└── 業界別ベンチマーク
    ├── 業界特有の成功パターン
    └── 業界別NGワード
```

## 3. AUTO - 自動取得（リアルタイム）

> **取得タイミング**: 資料生成時、リアルタイム
> **更新頻度**: 生成のたびに最新情報取得
> **取得元**: API連携、Webスクレイピング

### 3-1. CRM連携データ（既存顧客の場合）

```
CRM自動取得（AUTO）
├── 顧客情報
│   ├── 会社名
│   ├── 業界
│   ├── 従業員数
│   ├── 売上規模
│   └── 担当者情報
│
├── 商談情報
│   ├── 商談フェーズ
│   ├── 商談金額
│   ├── クローズ予定日
│   └── 競合情報
│
├── 活動履歴
│   ├── 過去の接点
│   ├── メール履歴要約
│   ├── 議事録要約
│   └── 過去提案資料
│
└── 契約情報（既存顧客）
    ├── 契約製品
    ├── 契約金額
    ├── 契約期間
    └── 利用状況
```

### 3-2. 企業公開情報

```
企業情報自動取得（AUTO）
├── 基本情報

│   ├── 本社所在地
│   ├── 設立年
│   ├── 代表者名
│   └── 従業員数
│
├── 財務情報（上場企業）
│   ├── 売上高（直近3年）

│   ├── 従業員数推移
│   └── 中期経営計画
│
├── ニュース・プレスリリース
│   ├── 直近6ヶ月のニュース
│   ├── プレスリリース
│   └── 経営方針発表
│
└── 採用情報
    ├── 求人情報
    ├── 組織拡大傾向
    └── 求める人材像
```

### 3-3. 業界情報

```
業界情報自動取得（AUTO）
```

## 4. USER - ユーザー入力（都度）

> **入力タイミング**: 資料生成時
> **入力者**: 営業担当者
> **UI**: CLI / Web フォーム

### 4-1. 必須入力項目

```
必須入力（USER - Required）
├── 顧客基本情報（CRM未連携 or 新規の場合）
│   ├── 会社名 *
│   ├── 業界 *（選択式）
│   └── 企業規模 *（選択式）
│
├── 商談コンテキスト
│   ├── プレゼン種類 *（選択式）
│   │   ├── 1次商談（課題仮説+ヒアリング）
│   │   ├── 2次商談（詳細提案）
│   │   ├── 月次定例
│   │   ├── アップセル提案
│   │   └── ...
│   │
│   ├── 今回のゴール *（自由記述 or 選択式）
│   │   └── 例: 「2次商談への移行承諾を得る」
│   │
│   └── プレゼン時間 *（選択式）
│       ├── 15分
│       ├── 30分
│       ├── 60分
│       └── カスタム
│
└── 課題仮説 *
    └── 顧客が抱えていると思われる課題（自由記述）
```

### 4-2. 任意入力項目

```
任意入力（USER - Optional）
├── 顧客詳細情報
│   ├── キーパーソン情報
│   │   ├── 氏名
│   │   ├── 役職
│   │   └── 関心事・性格
│   │
│   ├── 組織課題
│   │   ├── 経営課題
│   │   └── 部門課題
│   │
│   └── 過去のやり取り要約
│       └── 前回商談のポイントなど
│
├── 商談詳細
│   ├── 参加者情報
│   │   ├── 参加人数
│   │   ├── 役職構成
│   │   └── 初対面 or 既知
│   │
│   ├── 形式
│   │   ├── オンライン / 対面
│   │   ├── 投影 / 配布
│   │   └── モニターサイズ
│   │
│   └── 商談金額・タイムライン
│       ├── 想定金額
│       └── 導入希望時期
│
├── 競合状況
│   ├── 競合有無（単独 / コンペ）
│   ├── 競合企業名
│   └── この商談での差別化ポイント
│
└── 特別な要望・制約
    ├── 含めてほしい内容
    ├── 含めないでほしい内容
    ├── 特定スライドの指示
    └── トーン指定
```

## 入力負荷の最小化戦略

### 戦略1: CRM連携による自動補完

```
CRM連携時の入力削減
│
├── Salesforce / HubSpot 連携
│   └── 顧客情報、商談情報を自動取得
│       → ユーザー入力: ゴール、課題仮説のみ
│
└── 入力項目の削減効果
    ├── CRM未連携: 10項目入力
    └── CRM連携時:  3項目入力（70%削減）
```

### 戦略2: 選択式UIの活用

```
選択式による入力簡略化
│
├── 業界: ドロップダウン（25業界）
├── 企業規模: ラジオボタン（4段階）
├── プレゼン種類: ドロップダウン（10種類）
├── プレゼン時間: ラジオボタン（4段階）
└── ゴール: よく使うゴールのサジェスト
```

### 戦略3: 過去の商談からの引き継ぎ

```
引き継ぎ機能
│
├── 同一顧客の過去商談データを参照
├── 前回の課題仮説、フィードバックを自動反映
└── 商談の進行に応じたコンテキスト蓄積
```

### 戦略4: インテリジェントデフォルト

```
インテリジェントデフォルト
│
├── プレゼン種類 × 業界 → 推奨フレームワーク自動選択
├── 企業規模 × 業界 → 推奨事例自動選択
├── 過去の成功パターン → デフォルト構成を提案
└── 競合情報 → 差別化ポイントを自動生成
```
